Composer + Capistrano

Jul 12th, 2012

Background

After playing around with Ruby on Rails it was hard to comeback to PHP frameworks world. A lot of functionality was missing (assets, easy deployment, rake & etc). But most of all I missed Bundler system.

But recently I found Composer. And started messing around. Installation was very simple. It took me 5-10 min. integrating Zend Framework to existing project.

Problem

After this I started thinking how to integrate Composer into my deployment. I’m using Capistrano for deployment which is another great gem written in Ruby. Main problem which I was solving - when deploying project Capsitrano creates new directories from each deployment. So each time Composer installed libs over & over.

Solution

In the end I only had to write few lines of code & put them in deploy.rb file.

before "deploy:finalize_update", "deploy:remove_dirs"

namespace :deploy do
    desc "run composer install and ensure all dependencies are installed"
    task :install do
        run "ln -s #{deploy_to}vendor #{deploy_to}current/vendor"
        run "cd #{current_path} && php composer.phar install"
    end
end

I think code is self explanatory & I don’t have write what every line does. But if you have any questions - feel free to ask in comments.
